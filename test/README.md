# WebQuantumSavory Test Suite

This directory contains comprehensive tests for the WebQuantumSavory codebase.

## Test Files

- **`test_unit.jl`** - Unit tests that directly call functions from the WebQuantumSavory module
- **`test_integration.jl`** - Integration tests that test the API endpoints via HTTP requests
- **`test_simulation_integration.jl`** - End-to-end protocol simulations exercised through the HTTP API
- **`runtests.jl`** - Runs all tests (autogenerated)

## Test Data

The unit suite primarily uses `mock/payload.json`, while the integration suite uses
`mock/payload3.json`. Additional payloads cover more complex protocol configurations. The fixtures contain:
- 2 nodes (cities) with different slot configurations
- 1 edge connecting the nodes
- Various protocol definitions

## Running Tests

### Prerequisites

1. Make sure the main project dependencies are installed:
   ```bash
   julia --project=. -e 'using Pkg; Pkg.instantiate()'
   ```
2. Install test dependencies:
   ```bash
   julia --project=test -e 'using Pkg; Pkg.instantiate()'
   ```

### Run All Tests

```bash
julia --project runtests.jl
```

### Run Unit Tests Only

```bash
julia --project runtests.jl test_unit
```

### Run Integration Tests Only

**Note**: Integration tests require the server to be running on `http://localhost:8000`

```bash
# First start the server in another terminal
GENIE_ENV=test julia --project=. -e 'using WebQuantumSavory; WebQuantumSavory.main(); WebQuantumSavory.up(async=false)'

# Then run integration tests
julia --project runtests.jl test_integration test_simulation_integration
```

- **Base URL**: Integration tests use `http://localhost:8000` by default

- **Error Handling**: Tests verify both success and error conditions

## Adding New Tests

When adding new tests:

1. Use the `@safetestset` macro for test organization
2. Follow the existing naming conventions
3. Include both positive and negative test cases
4. Use the test payload data when appropriate
5. Ensure proper cleanup in integration tests
6. Add any new dependencies to `test/Project.toml`

## Troubleshooting

### Common Issues

1. **Server not running**: Integration tests require the server to be running
2. **Port conflicts**: Ensure port 8000 is available
3. **Dependency issues**: Run `Pkg.instantiate()` in the test directory
4. **Test conflicts**: Tests use unique names, but conflicts can occur if tests fail to clean up

### Debug Mode

To run tests with more verbose output, modify the logging configuration in the test runner files.
